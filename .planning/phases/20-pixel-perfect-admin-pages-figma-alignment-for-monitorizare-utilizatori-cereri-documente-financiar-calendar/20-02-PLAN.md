    **Step 2 — Create cereri-kanban-tab.tsx:**

    Props: `{ cereri: CerereRow[]; onStatusChange: (cerereId: string, newStatus: string) => void }`

    Renders 6 columns: depusa | verificare | info_supl | procesare | aprobata | respinsa.
    Each column: colored header badge + count + list of cerere cards.
    Cerere card: numar_inregistrare, priority indicator dot, SLA badge, functionar name.

    Click-to-move (no dnd-kit): clicking a cerere card opens a status-change dialog.
    Dialog: `fixed inset-0 z-[80] flex items-center justify-center bg-black/40`.
    Inside: 6 status buttons, current status highlighted. Selecting new status calls `onStatusChange`.
    `onStatusChange` in parent calls `updateCerereStatus` Server Action + updates local `cereri` state.

    When approving (selecting "aprobata"): also triggers confetti.

    Column max height: `max-h-[600px] overflow-y-auto`.
    Framer Motion: `AnimatePresence` on card list with `layout` prop, `motion.div` entry per card.

    **Step 3 — Create cereri-content.tsx (coordinator):**

    `"use client"`. Props: `{ cereri: CerereRow[]; functionari: FunctionarRow[] }`.

    State:
    - `activeTab: "overview" | "table" | "kanban" | "alerts"` (default "overview")
    - `localCereri: CerereRow[]` (initialized from cereri prop — mutated by local actions for optimistic UI)

    Header:
    - "Cereri Supervizare" title with FileText icon
    - Subtitle: "{total} cereri · {active} active"
    - Optional "Export" button (mock — shows toast.info "Funcționalitate în curând")

    4-tab navigation bar (same pattern as Figma):
    - Tabs: Overview | Tabel | Kanban | Alerte
    - Active tab: accent gradient bottom border indicator (motion.div layoutId)
    - Alerte tab: shows count badge if any cereri have SLA breach risk

    Tab content via AnimatePresence:
    - Overview → <CereriOverviewTab cereri={localCereri} />
    - Tabel → <CereriTableTab cereri={localCereri} functionari={functionari} onCerereAction={...} />
    - Kanban → <CereriKanbanTab cereri={localCereri} onStatusChange={handleStatusChange} />
    - Alerte → <CereriAlertsTab cereri={localCereri} />

    handleStatusChange: async (cerereId, newStatus) => {
      // Optimistic update
      setLocalCereri(prev => prev.map(c => c.id === cerereId ? { ...c, status: newStatus } : c));
      if (newStatus === "aprobata") confetti({ particleCount: 100, spread: 70, origin: { y: 0.6 } });
      const result = await updateCerereStatus(cerereId, newStatus);
      if (result.error) { toast.error(result.error); /* revert */ }
      else { toast.success("Status actualizat!"); }
    }
  </action>
