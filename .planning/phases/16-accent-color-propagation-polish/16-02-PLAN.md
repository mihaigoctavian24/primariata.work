  <name>Task 1: Create ClickableAvatar shared component</name>
  <files>src/components/shared/ClickableAvatar.tsx</files>
  <action>
Create a new "use client" component `ClickableAvatar` that provides click-to-upload avatar functionality. Reuse the upload logic from the existing `AvatarUpload.tsx` component (Supabase Storage upload to `user-avatars` bucket, file validation).

Props interface:
```typescript
interface ClickableAvatarProps {
  currentUrl: string | null;
  initials: string;
  size?: "sm" | "md" | "lg";  // sm=32px (top bar), md=48px (sidebar), lg=80px (settings)
  bucketPath?: string;  // e.g., "avatars" or "logos" -- defaults to "avatars"
  onUploadSuccess: (url: string) => void;
  className?: string;
}
```

Behavior:
- Renders a circular avatar: if `currentUrl` exists, show the image; otherwise show accent gradient background with white initials (use `var(--accent-gradient)` for fallback background)
- Hidden `<input type="file" accept="image/*">` triggered by click
- Hover state: semi-transparent dark overlay (bg-black/50) with Camera icon (lucide-react) centered
- On file select: validate (max 5MB, image/* type), upload to Supabase Storage `user-avatars` bucket at path `${bucketPath}/${userId}-${Date.now()}.${ext}`, get public URL, call `onUploadSuccess(url)`
- Loading state: show a small spinner overlay during upload
- Error state: show toast via sonner on failure
- Use Framer Motion for hover overlay animation (opacity fade)

Size mapping: sm=32px/text-xs, md=48px/text-sm, lg=80px/text-lg for the container and initials.

Import `createClient` from `@/lib/supabase/client` for the upload. Use the existing pattern from `AvatarUpload.tsx` for the storage upload logic -- do NOT duplicate the validation schema, import `avatarUploadSchema` if it's exported, otherwise create a simple inline validation.
  </action>
