**AdminLoginForm (`src/components/auth/AdminLoginForm.tsx`):**

Create a "use client" component with email + password login form for staff. Key differences from existing `LoginForm`:
- No Google OAuth button
- No "Remember me" checkbox
- No registration link (staff are invited)
- Post-login role check via `user_primarii` (not `utilizatori.rol`)
- Auto-detect primarie from `user_primarii` join

Zod schema:
```typescript
const adminLoginSchema = z.object({
  email: z.string().email("Adresa de email invalida"),
  password: z.string().min(8, "Parola trebuie sa contina minim 8 caractere"),
});
```

Form flow (onSubmit):
1. `supabase.auth.signInWithPassword({ email, password })`
2. On auth error: show translated error ("Email sau parola gresita")
3. On success: query `user_primarii` with join: `.select("primarie_id, rol, primarii(localitate_id, localitati(slug, judete(slug)))")` filtered by `.eq("user_id", data.user.id)` + `.in("rol", ["admin", "admin_primarie", "functionar", "primar", "super_admin"])` + `.eq("status", "approved")` + `.limit(1).single()`
4. If no staff association: call `supabase.auth.signOut()`, show error "Contul tau nu are acces la panoul de administrare." with redirect link "Esti cetatean? Selecteaza localitatea ta" linking to `/?step=2`
5. If staff: extract `judetSlug` and `localitateSlug` from the join, `router.push(/app/${judetSlug}/${localitateSlug}/admin)`

Form UI: Use React Hook Form + zodResolver. Two fields (email, password). "Ai uitat parola?" link points to `/auth/reset-password?return=admin`. Submit button text "Autentificare" with gradient background `var(--accent-gradient)`. Use motion.div for fade-in animation. Error display in `bg-destructive/10` alert box with optional redirect link.

**Auth callback role-aware redirect (`src/app/auth/callback/route.ts`):**

After the existing code that exchanges the auth code for a session and determines the redirect path, add a role check before the final redirect:

1. After session exchange, if user and primarie are determined, query `user_primarii` for the user's role: `.select("rol").eq("user_id", user.id).eq("primarie_id", primarie.id).eq("status", "approved").single()`
2. Use service role client (admin client) for this query since RLS might not be fully set up at callback time
3. If `rol` is one of `["admin", "admin_primarie", "functionar", "primar", "super_admin"]`, change `redirectPath` to `/app/${judetSlug}/${localitateSlug}/admin`
4. Keep existing redirect for non-admin roles (citizen dashboard)

**Middleware admin route isolation (`src/middleware.ts`):**

Add admin isolation logic: if user has an admin role in `user_primarii` for their current primarie, block access to citizen dashboard routes and redirect to admin dashboard.

Read the existing middleware carefully first. The middleware already has SEC-01/SEC-02 enforcement that blocks non-admins from `/app/*/admin/*`. Add the reverse: block admins from `/app/[judet]/[localitate]` when it's NOT an admin sub-path.

Implementation approach:
1. After the existing user session check, if the user is on a citizen route (`/app/[judet]/[localitate]` but NOT `/app/[judet]/[localitate]/admin/*`), check their role
2. If role is admin/staff (from the existing association check the middleware already does), redirect to `/app/[judet]/[localitate]/admin`
3. Be careful NOT to create redirect loops -- ensure `/app/[judet]/[localitate]/admin` does not trigger the citizen route check
4. Exclude shared routes that both roles need (e.g., `/app/[judet]/[localitate]/api/*` if any)

**Password reset return path:**

`ResetPasswordForm.tsx`: Read the component. Find the "Inapoi la autentificare" link that currently hardcodes `/auth/login`. Add a `?return=admin` query param check:
- Read `searchParams` using `useSearchParams()`
- If `return === "admin"`, the back link should point to `/admin/login`
- Otherwise keep `/auth/login`
- When calling `resetPasswordForEmail`, the `redirectTo` stays the same (`/auth/update-password`)

`UpdatePasswordForm.tsx`: Similarly, after password update success, the "Inapoi la autentificare" link should check if the original flow came from admin. The update-password page may receive a query param or you can check `document.referrer`. Simplest approach: add `?return=admin` to the `redirectTo` URL in ResetPasswordForm when return=admin, then UpdatePasswordForm reads it and links back to `/admin/login`.
