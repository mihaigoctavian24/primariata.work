    **Fix use-unread-notifications.ts:**

    The hook already has an initial fetch (`fetchUnreadCount`) and a realtime subscription. The issue from UAT is that the realtime subscription fails with CHANNEL_ERROR, and when it fails the count stays at 0. Looking at the code, the initial fetch IS there and SHOULD work independently of realtime. The likely issue is:

    1. The initial fetch may fail silently (RLS issue or user not yet loaded)
    2. The realtime subscription error may somehow reset the count

    Fix:
    1. Add error logging to the initial fetch to diagnose if it's failing
    2. Add a `try/catch` around the realtime subscription setup so CHANNEL_ERROR doesn't affect the initial count
    3. Add a polling fallback: if realtime subscription fails, set up a `setInterval` to refetch every 30 seconds as a degraded fallback
    4. Ensure the initial fetch runs BEFORE the subscription is set up, so even if subscription fails immediately, the initial count is preserved
    5. Add an `onError` handler to the channel subscription that logs the error but does NOT reset the count

    Updated structure:
    ```typescript
    export function useUnreadNotifications(userId: string | null, enabled: boolean = true) {
      const [unreadCount, setUnreadCount] = useState(0);

      useEffect(() => {
        if (!userId || !enabled) {
          setUnreadCount(0);
          return;
        }

        const supabase = createClient();
        let pollInterval: ReturnType<typeof setInterval> | null = null;

        const fetchUnreadCount = async () => {
          try {
            const { count, error } = await supabase
              .from("notifications")
              .select("*", { count: "exact", head: true })
              .eq("utilizator_id", userId)
              .is("read_at", null)
              .is("dismissed_at", null);

            if (!error && count !== null) {
              setUnreadCount(count);
            }
          } catch {
            // Silently fail -- keep previous count
          }
        };

        // Initial fetch
        fetchUnreadCount();

        // Try realtime subscription
        const channel = supabase
          .channel(`notifications-unread-${userId}`)
          .on("postgres_changes", {
            event: "*",
            schema: "public",
            table: "notifications",
            filter: `utilizator_id=eq.${userId}`,
          }, () => {
            fetchUnreadCount();
          })
          .subscribe((status) => {
            if (status === "CHANNEL_ERROR" || status === "TIMED_OUT") {
              // Fallback to polling when realtime fails
              if (!pollInterval) {
                pollInterval = setInterval(fetchUnreadCount, 30000);
              }
            }
          });

        return () => {
          supabase.removeChannel(channel);
          if (pollInterval) clearInterval(pollInterval);
        };
      }, [userId, enabled]);

      return unreadCount;
    }
    ```

    Also use a unique channel name per user (`notifications-unread-${userId}`) to avoid conflicts with the other notification subscription in `use-notifications-realtime.ts`.
