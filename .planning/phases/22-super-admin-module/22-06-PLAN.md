From src/middleware.ts (impersonation cookie reading to add):
Current middleware handles `/app/[judet]/[localitate]/*` routes for primarie context.
The impersonation cookie name: `sa_impersonation`.
Cookie value shape: `{ impersonating: true, targetPrimarieId: string }` (JSON string).
After reading the impersonation cookie in middleware, if targeting a super admin route (`/admin/primariata/*`):
  - Do NOT inject x-primarie-id (super admin is on their own panel)
If targeting a citizen/admin route (`/app/[judet]/[localitate]/*`):
  - The normal appRouteMatch block already resolves x-primarie-id from URL slugs
  - The impersonation cookie tells middleware to SKIP the user_primarii association check for this session
  - Add: if impersonation cookie present and current user is super_admin, bypass the association check

For the Impersonate button:
The CONTEXT.md says: "Super admin is the ONLY way to access /app/[judet]/[localitate]/admin — no direct navigation link from super admin panel. Mechanism: context flag cookie."

The impersonation flow:
1. Super admin clicks "Impersonează" in PrimarieDetailDrawer
2. Client calls `startImpersonation(primarieId)` server action
3. Server action: verifies super_admin role, fetches primarie localitate + judet slugs, sets cookie `sa_impersonation` via `cookies().set(...)` from `next/headers`, writes audit_log entry, returns { success: true, redirectUrl: `/app/${judetSlug}/${localitateSlug}/admin` }
4. Client receives redirectUrl and does `router.push(redirectUrl)`
5. Middleware sees the impersonation cookie on subsequent requests and injects x-primarie-id

Note: `cookies()` from `next/headers` in a server action sets cookies on the response. Use `import { cookies } from "next/headers"` and `(await cookies()).set("sa_impersonation", JSON.stringify({ impersonating: true, targetPrimarieId: primarieId }), { httpOnly: true, secure: true, sameSite: "lax", path: "/" })`.

DB columns needed for startImpersonation:
- `primarii`: id, localitate_id
- `localitati`: id, slug, judet_id
- `judete`: id, slug
- `audit_log`: insert actiune: "impersonate", detalii: { targetPrimarieId: primarieId }

**super-admin-write.ts — add startImpersonation action:**

Append `startImpersonation(primarieId: string): Promise<{ success: boolean; redirectUrl?: string; error?: string }>` to the file.

Implementation:
```typescript
import { cookies } from "next/headers";

export async function startImpersonation(primarieId: string): Promise<{ success: boolean; redirectUrl?: string; error?: string }> {
  try {
    // 1. Role check
    const supabase = await createClient();
    const { data: { user } } = await supabase.auth.getUser();
    if (!user) return { success: false, error: "Neautentificat" };
    const { data: userData } = await supabase.from("utilizatori").select("rol, prenume, nume").eq("id", user.id).single();
    if (!userData || userData.rol !== "super_admin") return { success: false, error: "Acces interzis" };

    // 2. Resolve primarie slugs
    const admin = createServiceRoleClient();
    const { data: primarieData } = await admin
      .from("primarii")
      .select("id, localitati!inner(slug, judete!inner(slug))")
      .eq("id", primarieId)
      .single();

    if (!primarieData) return { success: false, error: "Primărie negăsită" };

    // The join result shape — localitati is a single object (not array) for .single()
    const localitate = primarieData.localitati as { slug: string; judete: { slug: string } };
    const localitateSlug = localitate.slug;
    const judetSlug = localitate.judete.slug;

    // 3. Set impersonation cookie
    const cookieStore = await cookies();
    cookieStore.set("sa_impersonation", JSON.stringify({ impersonating: true, targetPrimarieId: primarieId }), {
      httpOnly: true,
      secure: process.env.NODE_ENV === "production",
      sameSite: "lax",
      path: "/",
      maxAge: 60 * 60 * 2, // 2 hours
    });

    // 4. Write audit_log
    const actorName = `${userData.prenume ?? ""} ${userData.nome ?? ""}`.trim();
    await admin.from("audit_log").insert({
      primarie_id: primarieId,
      utilizator_id: user.id,
      utilizator_nume: actorName,
      utilizator_rol: "super_admin",
      actiune: "impersonate",
      detalii: { targetPrimarieId: primarieId },
    });

    return { success: true, redirectUrl: `/app/${judetSlug}/${localitateSlug}/admin` };
  } catch (error) {
    logger.error("startImpersonation failed:", error);
    return { success: false, error: "A apărut o eroare" };
  }
}
```

Note: `utilizatori.nome` vs `utilizatori.nome` — check database.types.ts for exact column name. The project uses `prenume` and `nome` (confirmed in RESEARCH.md). If TypeScript rejects `nome`, use `nume` instead (check the exact column name).
