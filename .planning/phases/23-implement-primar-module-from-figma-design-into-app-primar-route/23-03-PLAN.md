**Read actions:**

1. `getPrimarDashboardData(): Promise<{ success: boolean; data?: PrimarDashboardData; error?: string }>`
   - getAuthContext(), fail if error
   - `supabase.from("user_primarii").select("id", { count: "exact", head: true }).eq("rol", "functionar").eq("status", "approved")` → functionariCount
   - `supabase.from("cereri").select("id, status, created_at, data_finalizare").is("deleted_at", null)` → all cereri for this primarie (RLS filters)
   - Compute from cereri array: cereriTotal, cereriLuna (created_at >= start of this month), cereriRezolvate (status=aprobata OR respinsa), satisfactiePct, timpMediu (avg days for finalizate cereri)
   - Group by YYYY-MM for last 6 months → cereriByMonth
   - `supabase.from("departamente").select("*")` → departamente
   - `supabase.from("proiecte_municipale").select("*").order("created_at", { ascending: false }).limit(5)` → proiecte (top 5 for dashboard)
   - Return `{ success: true, data: { ... } }`
   - If any query fails, return `{ success: false, error: "Eroare la încărcarea datelor dashboard." }`

2. `getPrimarCereriData(): Promise<{ success: boolean; data?: PrimarCereriData; error?: string }>`
   - getAuthContext()
   - escalatedQueue: `supabase.from("cereri").select("id, numar_inregistrare, status, created_at, data_finalizare, escaladata, prioritate, note_admin, note_primar, departament, tipuri_cereri(nume), utilizatori(nume, prenume)").eq("escaladata", true).is("deleted_at", null).order("created_at", { ascending: false })`
   - allCereri: same select without escaladata filter, limit 100
   - cereriByMonth: aggregate allCereri by YYYY-MM for last 6 months

3. `getPrimarBugetData(): Promise<{ success: boolean; data?: { departamente: DepartamentRow[]; totalBuget: number; totalConsumat: number }; error?: string }>`
   - getAuthContext()
   - `supabase.from("departamente").select("*").order("buget_alocat", { ascending: false })`
   - Also join proiecte_municipale for total buget: `supabase.from("proiecte_municipale").select("buget, buget_consumat")`
   - totalBuget = sum of all departamente.buget_alocat + sum of proiecte_municipale.buget
   - totalConsumat = sum of proiecte_municipale.buget_consumat (departamente don't track consumed)

4. `getPrimarProiecteData(): Promise<{ success: boolean; data?: { proiecte: ProiectRow[] }; error?: string }>`
   - getAuthContext()
   - `supabase.from("proiecte_municipale").select("*").order("created_at", { ascending: false })`

5. `getPrimarAgendeData(year: number, month: number): Promise<{ success: boolean; data?: { events: AgendaEventRow[] }; error?: string }>`
   - getAuthContext()
   - `supabase.from("agende_primar").select("*").eq("primar_id", userId)` filtered by data_eveniment between first and last day of the given month
   - Note: agende_primar RLS policy already filters by primar_id = auth.uid() — but explicit eq is fine for clarity

6. `getPrimarRapoarteData(period: "luna" | "trimestru" | "semestru"): Promise<{ success: boolean; data?: { cereriByMonth: { luna: string; total: number; rezolvate: number }[]; departamente: DepartamentRow[] }; error?: string }>`
   - getAuthContext()
   - Compute date range from period: last 1/3/6 months
   - `supabase.from("cereri").select("id, status, created_at, departament")` filtered by date range
   - Group by month + compute rezolvate per month
   - Also fetch departamente for performance table

7. `getPrimarSetariData(): Promise<{ success: boolean; data?: PrimarSetariData; error?: string }>`
   - getAuthContext()
   - `supabase.from("utilizatori").select("nume, prenume").eq("id", userId).single()` → displayName
   - `supabase.auth.getUser()` → email
   - `supabase.from("user_primarii").select("mandat_start, mandat_sfarsit").eq("user_id", userId).eq("status", "approved").single()` → mandat dates
   - Return assembled PrimarSetariData

Use `Promise.all()` where multiple independent queries run per action (Phase 19 pattern).

Handle all errors with try/catch — return `{ success: false, error: "Eroare la încărcarea datelor. Încearcă să recarci pagina." }` on any failure.

   - `supabase.from("cereri").update({ status: "aprobata", updated_at: new Date().toISOString() }).eq("id", cerereId)`
   - Also update `escaladata = false` (approved — no longer needs escalation)
   - `revalidatePath("/", "layout")` on success
   - Return `{}` on success, `{ error: err.message }` on failure

2. `rejectCerere(cerereId: string, motiv?: string): Promise<{ error?: string }>`
   - Same pattern, status = "respinsa", escaladata = false
   - If motiv provided, add to note_primar JSONB array (append pattern)

3. `addNotaPrimar(cerereId: string, text: string): Promise<{ error?: string }>`
   - getAuthContext()
   - Fetch current note_primar array: `supabase.from("cereri").select("note_primar").eq("id", cerereId).single()`
   - Append new entry: `[...existing, { text, timestamp: new Date().toISOString(), actor: userId }]`

   - revalidatePath + return

   - getAuthContext()
   - `supabase.from("proiecte_municipale").insert({ ...data, primarie_id: primarieId })`
   - revalidatePath + return

5. `updateProiect(id: string, data: Partial<Omit<ProiectRow, "id" | "primarie_id" | "created_at">>): Promise<{ error?: string }>`
   - getAuthContext()
   - `supabase.from("proiecte_municipale").update({ ...data, updated_at: new Date().toISOString() }).eq("id", id)`
   - revalidatePath + return

6. `deleteProiect(id: string): Promise<{ error?: string }>`
   - getAuthContext()
   - `supabase.from("proiecte_municipale").delete().eq("id", id)`
   - revalidatePath + return

**Agende CRUD:**

7. `createAgendaEvent(data: { titlu: string; data_eveniment: string; ora_start?: string; ora_sfarsit?: string; tip: AgendaEventRow["tip"]; descriere?: string }): Promise<{ error?: string }>`
   - getAuthContext()

   - getAuthContext()
   - `supabase.from("agende_primar").update(data).eq("id", id).eq("primar_id", userId)`

9. `deleteAgendaEvent(id: string): Promise<{ error?: string }>`
   - getAuthContext()
   - `supabase.from("agende_primar").delete().eq("id", id).eq("primar_id", userId)`
