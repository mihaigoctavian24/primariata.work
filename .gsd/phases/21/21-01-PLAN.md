# Plan 21.1: Admin Navigation Performance Optimization

## Objective

Fix the 1-3s delay when navigating between admin pages by adding `loading.tsx` skeletons, making the sidebar navigation optimistic, and moving blocking data queries into child Suspense boundaries.

## Context

- .gsd/ROADMAP.md
- .gsd/DECISIONS.md
- src/app/app/[judet]/[localitate]/admin/\*/page.tsx
- src/components/shell/sidebar/SidebarNavItem.tsx

<task type="auto">
  <name>Refactor Blocking DB Queries into Suspense Boundaries</name>
  <files>
    src/app/app/[judet]/[localitate]/admin/cereri/page.tsx
    src/app/app/[judet]/[localitate]/admin/utilizatori/page.tsx
    src/app/app/[judet]/[localitate]/admin/financiar/page.tsx
  </files>
  <action>
    - In these pages, the `page.tsx` currently does `await authClient.auth.getUser()`, verifies the admin role, and then immediately `await`s the heavy DB queries (cereri, functionari, plati, etc.).
    - Remove the heavy `await db_queries()` from `page.tsx`.
    - Create a wrapper Server Component (e.g. `<CereriDataWrapper primarieId={primarieId} />`) that performs the heavy query and passes the data to the client `Content` component.
    - In `page.tsx`, return `<Suspense fallback={<TheLoadingSkeleton />}><CereriDataWrapper primarieId={primarieId} /></Suspense>`.
    - This allows `page.tsx` to quickly return the UI after the auth check, streaming the heavy data once ready.
  </action>
  <verify>grep -n "Suspense" src/app/app/\[judet\]/\[localitate\]/admin/cereri/page.tsx</verify>
  <done>User navigating to /admin/cereri sees the skeleton instantly while Supabase fetching runs in parallel.</done>
</task>
